Promises

This module provides the "force" command which evaluates a script and then saves the result inside the object so that next time it's called with the same object it can give a result without evaluating the script again.

Note that the usage of the word "promise" here follows the terminology of Scheme's lazy evaluation facility, and has nothing to do with concurrency.

Good practice suggests only using this command on idempotent scripts with no side effects.
